# Гайд по переводу

### Названия и заголовки

Название файла всегда должно быть написано маленькими буквами. Если слов несколько, использовать дефис. Это нужно для корректного формирования ссылок на сайте. 

Пример: That'll Show ‘Em → thatll-show-em

Заголовок заметки всегда должен выглядеть так:

```
---
title: Название (Name)
---
# Название (Name)
```

Это требуется для корректного отображения внутри хранилища и на сайте. [[zone-of-community/translation#Вставки|Подробнее тут]].

### Двойные пробелы

Если строки идут подряд, они должны оканчиваться на 2 пробела, чтобы на сайте с сливаться в одну строку.

Проверить можно плагином Show Whitespace.

В остальном абзацы разделяются через одну пустую строку.

### Режимы

Obsidian предоставляет различные режимы работы. Наиболее частые это режим исходного кода и динамический режим. Режим просмотра нужен только если хочется посмотреть, как оно будет выглядеть в конечном итоге.

Переключение между режимами происходит через `Ctrl + e` и `Ctrl + r`. Текущий режим отображается внизу справа.

Рекомендуется создавать документ в режиме исходного кода, а проверять результат и вносить правки в динамическом режиме.

### Ссылки

В правилах много [[zone-of-community/translation#Ссылки|ссылок]].

Чтобы создать ссылку на заметку, откройте 2 квадратные скобки и начните писать название заметки, title заметки на русском или на английском. Система предложит варианты, выберите нужный и нажмите Enter. Некоторые варианты показываются 2 раза, можно выбирать любой, это одна и та же заметка.

Например: открываем 2 квадратные скобки, пишем ar и первым вылетает arc, не дописывая жмём Enter и получаем ссылку формата [[rules/traits/arc|Дуга (Arc)]].

Чтобы изменить текст, ссылки, отображаемый в динамическом режиме, нужно изменить текст внутри ссылки после вертикальной черты `|` например [[zone-of-community/translation|так]].

Чтобы сделать ссылку на конкретный абзац, нужно использовать `#` после названия заметки: [[zone-of-community/translation#Ссылки|Гайд по переводу]]. В этом случае по название ссылки не изменится и будет всё равно ссылаться на название заметки. Его необходимо изменить вручную.

Наиболее простой способ создания таких ссылок: открыть квадратные скобки, начать писать название заметки, выбрать из предложенного списка заметку и до нажатия Enter нажать `#`. После чего ссылка предложи выбрать нужный абзац. Можно выбрать нужный или начать писать его название и подтвердить, нажав Enter.

#### Вставки

Иногда нужно сделать ссылку, которая отображает информацию из другой заметки. Для этого нужно использовать `!` перед ссылкой. Например так:

![[zone-of-community/translation#Двойные пробелы]]

Такие ссылки отображаются только в динамическом режиме и режиме просмотра.

Также такие ссылки не требуют текст после `|`, который рекомендуется удалить.

Ссылка на заметку целиком, требует ссылаться на его заголовок первого уровня:

Неправильно:

![[rules/activities/all-activities/step]]

Правильно:

![[rules/activities/all-activities/step#Шаг (Step) ◆]]


#### Абсолютные пути

При вставке ссылок через
```
[[
```

иногда появляется модальное окно, предлагающее использовать короткий вариант ссылок формата

```
Approve changes for <название текущего файла>

- [[name]] => [[name|Название]]
```

Система пытается создать короткую ссылку к заметке. Это может привести к сложностям в будущем. Всегда используйте абсолютные пути. 

Когда появляется такая модалка (обычно при указании полного имени файла), вежливо откажите системе, нажав `Cancel` или Escape. Если же вы нажали Apply или Enter, просто сотрите ссылку.

```
[[arc|Дуга (Arc)]] — неправильно

[[rules/traits/arc|Дуга (Arc)]] — правильно

```

Чтобы минимизировать случаи появления модалки, не пишите название целиком при создании ссылки, а выберите нужное из предлагаемых вариантов.
